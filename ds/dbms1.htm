1.1 DATABASE SYSTEM CONCEPTS AND ARCHITECTURE

What is a Database System?

Database System (DBS)

An integrated collection of related data along with a set of programs to access, manage, and control the data. It is designed to handle large amounts of information by storing, retrieving, and managing data efficiently.

Components of a Database System:
  • Database (DB): Organized collection of logically related data
  • Database Management System (DBMS): Software system that enables users to define, create, maintain, and control access to the database
  • Database Applications: Programs that interact with the database through the DBMS
  • Users: End users, application programmers, database administrators (DBAs)
  • Hardware: Physical devices (computers, storage, network)
  • Procedures: Instructions and rules for using the database system

Data Models: The Blueprint of Databases

Data Model

A collection of concepts for describing data, data relationships, data semantics, and consistency constraints.

Categories of Data Models:
  • Conceptual (High-level) Data Models: Close to how users perceive data. Examples: Entity-Relationship (ER) model, Object-Oriented model.
  • Physical (Low-level) Data Models: Describe how data is stored on computer storage media. Examples: File structures, B-trees, Hash indexes.
  • Implementation (Representational) Data Models: Between conceptual and physical models. Examples: Relational model, Network model, Hierarchical model.
Historical Evolution of Data Models:

Three-Schema Architecture (ANSI/SPARC Architecture)

Proposed in 1975 to achieve data independence, this architecture separates the user applications from the physical database.

The Three Levels:
Key Benefit: Data Independence

Logical Data Independence: Ability to change the conceptual schema without having to change external schemas or application programs.

Physical Data Independence: Ability to change the internal schema without having to change the conceptual schema.

Database Languages and Interfaces

Database Languages:
Language Purpose Key Commands
Data Definition Language (DDL) Define the database schema CREATE, ALTER, DROP, TRUNCATE, RENAME
Data Manipulation Language (DML) Manipulate data in the database SELECT, INSERT, UPDATE, DELETE
Data Control Language (DCL) Control access to data GRANT, REVOKE
Transaction Control Language (TCL) Manage transactions COMMIT, ROLLBACK, SAVEPOINT
Example DDL and DML:
DDL Example:
CREATE TABLE Student (
    StudentID INT PRIMARY KEY,
    Name VARCHAR(50) NOT NULL,
    Major VARCHAR(30),
    GPA DECIMAL(3,2)
    );
DML Example:
INSERT INTO Student VALUES (104, 'Alice Brown', 'Biology', 3.7);
SELECT Name, GPA FROM Student WHERE GPA > 3.5;
UPDATE Student SET GPA = 3.9 WHERE StudentID = 101;
1.2 CONCEPTUAL DATA MODELING USING ENTITY-RELATIONSHIP (ER) MODEL

Introduction to ER Modeling

The Entity-Relationship (ER) Model is a high-level conceptual data model developed by Peter Chen in 1976. It provides a graphical representation (ER diagrams) for database design.

Entity Types and Attributes

Types of Attributes:
Key Types:
  • Superkey: Set of one or more attributes that uniquely identifies an entity
  • Candidate Key: Minimal superkey (no proper subset is a superkey)
  • Primary Key: Candidate key chosen by database designer as main identifier
  • Foreign Key: Attribute(s) in one relation that reference primary key of another relation
