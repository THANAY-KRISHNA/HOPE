1. BASIC STRUCTURE

4. CONDITIONALS

if-else

    // ...

    // ...

8. METHODS

[access] [static] returnType methodName([params]) {
    // body
    return value; // if returnType ‚â† void
}

// Example

    return a + b;
}

// Call

üîë Key points:

  • static methods can be called without object: ClassName.method()
  • Overloading: same name, different params.

9. CLASSES & OBJECTS

    // fields
    dataType fieldName;

    // constructor
    ClassName([params]) {
        // initialize fields (use 'this' if shadowing)
        this.fieldName = fieldName;
    }

    // methods
    dataType methodName() { ... }

// Object creation

10. CONSTRUCTOR & this

class Student {
    String name;
    int roll;

    Student(String name, int roll) {
        this.name = name;   // 'this' resolves shadowing
        this.roll = roll;
    }

    Student() {
        this("Unknown", 0); // constructor chaining
    }
}

11. INHERITANCE

    // override method
    @Override
    void methodName() { ... }
}

‚úÖ Use super() to call parent constructor (must be first line).

12. POLYMORPHISM

Overriding (Runtime)

Parent p = new Child();  // upcasting
p.method();              // ‚Üí Child's version (if overridden)

Dynamic Dispatch

JVM calls method based on object type, not reference type.

13. ABSTRACTION

Abstract Class

    abstract double area();   // no body
    void common() { ... }     // concrete allowed
}

Interface

interface Drawable {
    void draw();               // public abstract by default
    default void info() { ... } // Java 8+
    static void help() { ... }
}

class Circle implements Drawable {
    public void draw() { ... } // 'public' mandatory
}

‚úÖ A class can extends one class, implements many interfaces.

16. EXCEPTION HANDLING

    // risky code (e.g., file/db/io)
} catch (SpecificException e) {
    // handle
} catch (Exception e) {
    // fallback
} finally {
    // cleanup (e.g., close resources)
}

// Throw custom exception

// Declare exception

‚úÖ finally runs even if return/break.

    // use fr
} catch (IOException e) { ... }

18. JDBC (4-Step Template)

import java.sql.*;

// 1. Connection
Connection con = DriverManager.getConnection(
    "jdbc:mysql://localhost/db", "user", "pass");

// 2. Statement
Statement stmt = con.createStatement();

// 3. Execute
ResultSet rs = stmt.executeQuery("SELECT * FROM table");

// 4. Process
while (rs.next()) {
    int id = rs.getInt("id");          // by column name
    String name = rs.getString(2);     // by index (1-based)
}

// Close
rs.close(); stmt.close(); con.close();

‚úÖ For inputs: use PreparedStatement to avoid SQL injection.

21. CUSTOM EXCEPTION

class MyException extends Exception {
    MyException(String msg) {
        super(msg);
    }
}

// Usage
void check(int x) throws MyException {
    if (x < 0) throw new MyException("Negative!");
}

Extend Exception ‚Üí checked; extend RuntimeException ‚Üí unchecked.

22. SINGLETON PATTERN (Thread-Safe)

class Singleton {
    private static Singleton instance;

    private Singleton() {} // private constructor

    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null)
                    instance = new Singleton();
            }
        }
        return instance;
    }
}

‚úÖ getInstance() is the only way to access.
